## ⚙️ Service Dependencies

### Discovery Server
- **Must start first** - All other services depend on it
- Wait ~30 seconds for full startup

### Config Server
- **Starts second** - Provides configuration to other services
- Wait ~30 seconds for full startup

### API Gateway
- **Starts third** - Routes requests to backend services
- Wait ~25 seconds for full startup

### Business Services (Event, User, Booking)
- Can start in parallel after infrastructure services
- Wait ~15-20 seconds each for full startup

### Frontend
- Can start anytime after backend services are up
- Requires backend to be healthy for full functionality

## 🐛 Troubleshooting

### Service Not Starting
```bash
# Check if port is already in use
lsof -ti:<port>

# Kill process on port
lsof -ti:<port> | xargs kill -9

# Check logs for errors
tail -100 logs/<service-name>.log | grep -i error
```

### Service Not Registering with Eureka
- Wait 30-60 seconds for registration
- Check Discovery Server is running
- Verify `application.yml` eureka configuration
- Check logs for connection errors

### Frontend Compilation Errors
```bash
# Reinstall dependencies
cd frontend
rm -rf node_modules package-lock.json
npm install
npm run dev
```

### Database Issues
- Currently using H2 in-memory database
- Data is lost on service restart
- Check `application.yml` for datasource configuration

## 🔄 Development Workflow

### Making Changes to Backend
1. Stop the affected service: `lsof -ti:<port> | xargs kill -9`
2. Make your changes
3. Rebuild: `cd <service-directory> && mvn clean compile`
4. Restart: `mvn spring-boot:run > ../../logs/<service>.log 2>&1 &`

### Making Changes to Frontend
1. Changes are auto-reloaded (Hot Module Replacement)
2. If needed, restart: 
   ```bash
   lsof -ti:3000 | xargs kill -9
   cd frontend && npm run dev > ../logs/frontend.log 2>&1 &
   ```
